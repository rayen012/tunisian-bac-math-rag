\section{Conclusion}
\label{sec:conclusion}

% ─────────────────────────────────────────────────────────────────────────────
\subsection{Summary}
\label{subsec:summary}

This thesis presented a RAG system for the Tunisian Baccalaureate
mathematics examination. We built a complete pipeline---from document
acquisition and multimodal OCR through adaptive chunking and vector
indexing to bilingual answer generation---and compared three variants: a
RAG system grounded in retrieved documents, a prompt-only baseline encoding
the syllabus in the system prompt, and a hybrid engine routing between the
two based on retrieval quality.

A blind evaluation by a Tunisian Bac mathematics teacher produced a result
counter to our initial expectations: the Prompt-Only baseline (4.32/5)
outperformed both the RAG system (3.22/5) and the Hybrid engine (3.25/5)
on every criterion. All three systems passed all guardrail tests. The
digitization pipeline achieved 5.4\% CER and 97.1\% \LaTeX{} math accuracy.

The key takeaway is that for a domain where the LLM's parametric knowledge
already covers the mathematical content, and where the retrieval corpus is
modest in scale, careful prompt engineering can outperform retrieval
augmentation. The value of RAG depends on whether the corpus is large and
precise enough to consistently beat what the model already knows.

\subsection{Future Work}
\label{subsec:future-work}
\label{sec:future-work}

\begin{enumerate}
    \item \textbf{Corpus expansion.} The most direct test of whether
    retrieval can surpass prompt engineering: significantly more exams,
    exercise series, and corrections across all chapters. A denser corpus
    would increase the chance of retrieving closely matching corrections.

    \item \textbf{Multi-rater evaluation.} Repeating the blind evaluation
    with multiple teachers would allow inter-rater reliability analysis. A
    sample of 30--50 questions scored by at least two evaluators would
    permit significance testing.

    \item \textbf{Student user study.} The current evaluation measures
    answer quality as judged by a teacher. A study with actual Bac students
    could assess whether the system improves learning outcomes.

    \item \textbf{Embedding fine-tuning.} Fine-tuning BGE-M3 on Tunisian
    Bac text could improve chapter-level retrieval precision without
    requiring a larger corpus.

    \item \textbf{Automated evaluation.} LLM-as-judge metrics for
    mathematical correctness and style adherence would enable larger-scale
    evaluation without the bottleneck of manual grading.

    \item \textbf{Interactive tutoring.} Adding step-by-step guided problem
    solving---where the system reveals the solution incrementally and
    responds to student attempts---would move it from a correction generator
    toward an interactive tutor.
\end{enumerate}
