\section{Evaluation Methodology}
\label{sec:evaluation-methodology}
\label{sec:eval-setup}

Automated metrics such as BLEU or ROUGE are not well suited to this task,
because they do not capture mathematical validity, quality of explanation,
or adherence to Bac-specific writing conventions~\cite{kasneci2023chatgpt_education}.
For this reason, the evaluation was based on blind human judgment by a
domain expert.

\subsection{Question Bank}

The evaluation used 20 questions divided into five categories. Category~A contains five
direct Bac-style exercises that were likely to have close matches in the
corpus and were mainly intended to test retrieval precision. Category~B also
contains five questions, but these are novel exercises on Bac topics with
different formulations, so they test generalization beyond memorized
patterns. Category~C contains three informal student-style questions with
incomplete phrasing or colloquial French, which makes them useful for
evaluating coaching quality. Category~D contains two questions written in
Tunisian Derja or mixed language and was included to test linguistic
robustness. Finally, Category~E contains four out-of-scope questions using
methods such as L'H\^opital, diagonalization, Taylor expansion, or Lebesgue
integration. These were not graded on the main rubric, but used as guardrail
tests with pass/fail evaluation.

Only Categories~A to~D, for a total of 15 questions, were included in the
graded comparison.

\subsection{Grading Rubric}

Each of the 15 graded questions was evaluated for every system on four
criteria, using a 0--5 scale with equal weighting. The criteria were
mathematical correctness, reasoning clarity, pedagogical quality, and
adherence to Bac style. Together, these dimensions were intended to capture
not only whether an answer was correct, but also whether it was useful for a
student preparing for the Tunisian Baccalaureate.

\subsection{Blind Protocol}
\label{subsec:blind-evaluation}

For each question, the three system outputs were randomly shuffled and shown
under anonymous labels (Syst\`{e}me~X, Y, and Z). The evaluator, an
experienced Tunisian Bac mathematics teacher, did not know which system
produced which answer. The answer sheets were generated automatically, and
the system identities were revealed only after the grading had been
completed.

\subsection{Example~1: Low Distance, Wrong Content (B02---Probabilités)}

\noindent\textbf{Query:} \textit{``Urne: 5 blanches, 3 noires. Tirage
successif sans remise de 3 boules. Calculer $P$(exactement 2 blanches).''}

This query had the lowest retrieval distance in the evaluation
($d = 0.3989$), yet the retrieved context was not a good topical match. The
RAG system produced a short answer based on generic retrieved sources. The
prompt-only system correctly recognized the exercise type and gave a more
complete solution with explicit combinatorial reasoning. The hybrid system
also relied on retrieved context, but part of that context corresponded to a
different variant of the problem.

\noindent\textbf{Takeaway:} This example shows that a low embedding distance
did not necessarily mean that the retrieved material was the right one for
the question. In this case, retrieval seems to have constrained the answer
instead of helping it.

\subsection{Example~2: Correct Chapter Match (D01 --- Suites réelles)}

\noindent\textbf{Query (Derja):} \textit{``Kifech nethbet elli suite
monotone et bornée convergente?''}

This was one of the cases where retrieval matched the expected chapter.
All three systems produced relevant answers, and the difference in quality
between them was smaller than in Example~1. This is consistent with the
quantitative results, where Category~D showed a narrower gap than the other
categories. The example suggests that retrieval is more useful when the
retrieved context is genuinely aligned with the question.

\subsection{Example~3: Subtopic Mismatch (C01 --- Continuité)}

\noindent\textbf{Query:} \textit{``Je comprends pas comment on étudie la
continuité d'une fonction en un point.''}

Here the student asks about continuity at a point. The retrieval returned
material related to continuity, but focused on continuity over intervals
rather than the specific pointwise case. Although this still counted as a
chapter match, the retrieved context did not fully answer the student's
actual need. The RAG system adapted the available material, but did not give
the clearest pointwise explanation. The prompt-only system, by contrast,
stated the definition directly and illustrated it with a more appropriate
example.

The hybrid engine partially recognized the mismatch, but since the query was
still classified as a strong retrieval case, it did not fully shift away
from the retrieved context.

\textbf{Pattern.} These examples show two different failure modes. In some
cases, retrieval returns material that is close in embedding space but not
topically precise enough. In others, it finds the correct chapter but still
misses the exact subtopic needed for the question. When retrieval is well
aligned, the gap between systems becomes smaller. When it is only
approximately aligned, it can reduce answer quality instead of improving it.

system would have performed better end to end.

\section{Digitization Quality}
\label{sec:ocr-evaluation}

material, including dense mathematical notation, text-heavy proofs, course

\textbf{Metrics.} Three metrics were used: CER (character error rate),

retrieval-based systems, not the prompt-only baseline.
